# Performance & SEO Optimization Plan\n\n## 1. Audit Findings\n- **Performance Score: 74** (Target: 90+)\n- **Accessibility Score: 90** (Target: 100)\n- **SEO Score: 91** (Target: 100)\n- **LCP:** 5.1s (Too slow, caused by heavy JS bundle and unoptimized hero image/fonts).\n- **FCP:** 3.4s (Caused by render-blocking Google Fonts).\n- **Bundle Size:** Large JS payload due to lack of code splitting (ReactQuill loaded on the main page).\n- **SEO Error:** `vercel.json` wildcard rewrite is breaking `robots.txt`, serving the HTML index instead of the text file.\n\n## 2. Execution Strategy\n\n### Step 1: SEO Fixes (Robots & Vercel Config)\n- Create a proper `public/robots.txt`.\n- Update `vercel.json` to exclude files with extensions from the SPA rewrite so `robots.txt` and `sitemap.xml` resolve correctly.\n\n### Step 2: Font Optimization (Fix FCP)\n- Update `index.html` to use `preconnect` and `display=swap` more effectively for Google Fonts.\n- Preload the critical hero background image to fix LCP.\n\n### Step 3: Bundle Optimization (Fix LCP & JS Payload)\n- Implement React `lazy` and `Suspense` in `App.tsx` to code-split the Admin Dashboard and Blog Editor.\n- This ensures visitors to the landing page don't download the heavy `react-quill` library.\n\n### Step 4: Accessibility Fixes\n- Ensure the mobile menu trigger button has an `aria-label`.\n- Ensure form inputs have proper accessible names.\n\n## 3. Verification\n- Run `npm run build` and check the CLI output for chunk sizes.\n- Re-run the Lighthouse audit locally or via CLI to verify Performance > 90.
