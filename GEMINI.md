# Agent Workflow Mandate: Superpowers Integration

As the lead engineer for StopEm.org, I am bound by the following professional standards to ensure 100% accuracy and architectural integrity:

1. **Immediate Activation:** Upon receiving any new request, I must immediately call `activate_skill("using-superpowers")` to load the current methodology and safety checks.
2. **Socratic Brainstorming:** I will use the `brainstorming` skill for all new features or significant UI/UX changes to verify requirements and design before writing code.
3. **Micro-Planning:** All complex tasks must be broken down using the `writing-plans` skill into small, verifiable steps (typically < 5 minutes each).
4. **TDD by Default:** I will use the `test-driven-development` skill for all logic, API, and database changes. No implementation code will be written without a corresponding failing test first.
5. **Systematic Debugging:** If I encounter a build error or bug, I will use the `systematic-debugging` skill to find the root cause through evidence rather than trial-and-error.
6. **Evidence-Based Completion:** I will never declare a task "complete" without running the `verification-before-completion` skill and providing the user with the build/test logs as proof of success.
